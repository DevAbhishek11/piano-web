# Prompt: Build a Production-Ready Realistic Web Piano App

## Short Summary

Build a beautiful, realistic, easy-to-use web piano app using **Next.js, React, TypeScript, and Web Audio API**.

The app should let users play piano with:

- Computer keyboard
- Mouse
- Touch screen
- Multi-touch chords

It should also include:

- Realistic piano sound
- Sustain pedal
- Recording and playback
- Metronome
- Lessons/practice mode
- Local user profiles
- Import/export profile data
- Themes
- Responsive mobile-friendly design

The final result should be polished enough to deploy as a real standalone piano learning and playing website.

---

# 1. Role

You are an expert frontend engineer, audio engineer, React/Next.js developer, and UI/UX designer.

Create a complete production-grade piano application with clean code, strong architecture, realistic sound, and a simple user experience.

Focus on making the app:

- Easy to use
- Beautiful
- Fast
- Realistic
- Beginner-friendly

- Production-ready

# 5. Piano Layout

Create a realistic piano keyboard with:

- White keys
- Black keys
- Correct piano arrangement
- At least 3 octaves
- Preferably from `C3` to `C7`

Notes must include:

```txt
C, C#, D, D#, E, F, F#, G, G#, A, A#, B
```

Correct black key placement:

```txt
C# between C and D
D# between D and E
F# between F and G
G# between G and A
A# between A and B
```

There should be no black keys between:

```txt
E and F
B and C
```

Use generated note data instead of hardcoding every key manually.

Example note object:

```ts
type PianoNote = {
  note: string;
  octave: number;
  fullNote: string;
  frequency: number;
  type: 'white' | 'black';
  keyboardLabel?: string;
  keyboardCode?: string;
  shiftKeyboardLabel?: string;
};
```

# 8. Realistic Piano Sound

Make the sound as close as possible to a real acoustic piano.

## Sound Requirements

The piano sound should include:

- Natural attack
- Warm body tone
- Realistic decay
- Long natural release
- Velocity response
- Sustain resonance
- Soft stereo width
- Slight hammer character
- Natural damping when notes are released
- Polyphonic playback
- Smooth repeated notes without harsh cutting

## Preferred Audio Approach

Use real piano samples if available.

Sample folder:

```txt
/public/samples/piano/
```

Sample naming format:

```txt
C3.mp3
Cs3.mp3
D3.mp3
Ds3.mp3
E3.mp3
F3.mp3
Fs3.mp3
G3.mp3
Gs3.mp3
A3.mp3
As3.mp3
B3.mp3
...
C7.mp3
```

If real samples are missing, use a high-quality synthesized fallback.

The fallback should use:

- Multiple oscillators
- Slight detuning
- Harmonic layering
- ADSR envelope
- Exponential decay
- Low-pass filter
- Gentle high-frequency roll-off
- Subtle reverb
- Optional impulse-like hammer click
- Optional stereo panning

The fallback should sound piano-like, not like a basic beep.

## Audio Engine API

Create a browser-only audio engine separate from React.

Example:

```ts
class AudioEngine {
  init(): Promise<void>;
  resume(): Promise<void>;
  preloadSamples(notes: string[]): Promise<void>;
  noteOn(note: string, velocity?: number): string;
  noteOff(voiceIdOrNote: string): void;
  setSustain(enabled: boolean): void;
  setVolume(value: number): void;
  setReverb(amount: number): void;
  setToneBrightness(value: number): void;
  stopAll(): void;
}
```

Important:

- `noteOn` should return a unique voice ID.
- The same note should be playable repeatedly without killing the previous voice instantly.
- Audio state should be managed outside React rendering.
- Resume AudioContext after first user interaction.
- Clean up old voices automatically.

- Mobile multi-touch behavior
